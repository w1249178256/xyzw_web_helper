# 十殿 8 功能说明

## 📋 功能概述

在十殿信息卡片的 teamid 子卡片中新增了"十殿 8"按钮，用于自动执行十殿 8 的加入流程。

## 🎯 功能位置

```
十殿信息卡片 → 十殿 8 按钮
位置：在"加入房间"按钮之后，"开始十殿"按钮之前
```

## 🚀 功能说明

### 界面元素

```
┌─────────────────────────────────────┐
│  十殿 8  [输入框：输入队伍号]  [按钮] │
└─────────────────────────────────────┘
```

- **按钮名称**：十殿 8
- **输入框**：用于输入队伍号（teamId）
- **位置**：加入房间按钮后面

### 执行流程

#### 第一步：处理 8#7 Token
```
1. 查找 1 个名称带 "8#7" 的 Token
2. 连接游戏（模拟点击 Token 昵称）
3. 获取十殿枕头数量
4. 如果枕头数量 = 5：
   → 模拟点击"加入十殿"按钮
   → 使用输入的队伍号加入
5. 如果枕头数量 ≠ 5：
   → 提示错误并停止
```

#### 第二步：处理 3 个 8# Token
```
1. 查找 3 个名称带 "8#" 的 Token（排除 8#7）
2. 依次处理每个 Token：
   → 连接游戏（模拟点击 Token 昵称）
   → 获取十殿枕头数量
   → 如果枕头数量 = 5：
      * 模拟点击"加入十殿"按钮
      * 使用输入的队伍号加入
   → 如果枕头数量 ≠ 5：
      * 提示并跳过
   → 等待 1 秒（除了最后一个）
```

## 📝 使用步骤

### 1. 准备 Token
确保有以下命名的 Token：
- 至少 1 个 Token 名称包含 `8#7`
- 至少 3 个 Token 名称包含 `8#`（可以是 8#1, 8#2, 8#3 等）

### 2. 输入队伍号
在"十殿 8"按钮后的输入框中输入队伍号（teamId）

### 3. 执行
点击"十殿 8"按钮，系统将自动执行以下操作：

```
输入队伍号 → 点击"十殿 8"按钮
    ↓
查找 8#7 Token
    ↓
检查枕头数量 = 5？
    ↓ 是
加入十殿（使用输入的队伍号）
    ↓
查找 3 个 8# Token
    ↓
依次检查枕头数量 = 5？
    ↓ 是
依次加入十殿
    ↓
执行完成
```

## ⚠️ 注意事项

### 1. Token 命名要求
- 必须包含 `8#7` 的 Token（至少 1 个）
- 必须包含 `8#` 的 Token（至少 3 个，不包括 8#7）

示例命名：
```
8#7
8#1
8#2
8#3
```

### 2. 枕头数量要求
- 所有 Token 的十殿枕头数量必须为 5
- 如果枕头数量不为 5，会跳过该 Token 或停止执行

### 3. 队伍号
- 必须输入有效的队伍号
- 队伍号必须是数字

### 4. 错误处理
- 如果找不到对应的 Token，会提示并停止
- 如果枕头数量不为 5，会提示并跳过或停止
- 如果加入失败，会显示错误信息

```
总执行时间：约 8-12 秒
```

## 🔄 与其他功能的区别

### 与"加入房间"的区别
- **加入房间**：手动加入，需要手动选择 Token 和输入队伍号
- **十殿 8**：自动执行，自动查找 Token 并批量加入

### 与"批量十殿"的区别
- **批量十殿**：完整的十殿流程（殿 7、殿 2、殿 5、殿 0）
- **十殿 8**：专门的 8 号队伍流程（8#7、8#）

## 💡 使用场景

### 场景 1：多账号同时刷十殿
当有多个账号（8#7, 8#1, 8#2, 8#3）需要同时刷十殿时，使用此功能可以快速将所有账号加入同一个队伍。

### 场景 2：固定队伍刷十殿
当有固定的 4 人队伍，且都使用 8 号命名规则时，可以快速组建队伍。

### 场景 3：重复性操作自动化
避免手动逐个点击加入房间的重复操作，一键完成。

## 📞 故障排查

### 问题 1：提示"未找到名称带 8#7 的 Token"
**解决方案**：
- 检查 Token 名称是否正确
- 确保至少有一个 Token 名称包含 `8#7`

### 问题 2：提示"枕头数量不为 5"
**解决方案**：
- 先执行"获取"按钮获取枕头数量
- 确保 Token 的十殿枕头数量为 5

### 问题 3：加入房间失败
**解决方案**：
- 检查队伍号是否正确
- 检查房间是否已满
- 检查网络连接状态

### 问题 4：执行时间过长
**解决方案**：
- 检查 Token 数量是否正确
- 检查网络连接
- 确认没有 Token 被占用
