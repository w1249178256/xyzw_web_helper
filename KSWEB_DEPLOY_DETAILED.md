# 🌐 KSWEB 部署详细指南

## 📋 概述

**KSWEB** 是 Android 平台上最简单的 Web 服务器解决方案，无需 Root 权限，图形化界面操作，适合所有用户。

### ✨ 优势
- ✅ **无需 Root** - 普通手机即可使用
- ✅ **图形界面** - 直观易用，无需命令行
- ✅ **一键启动** - 简单配置即可运行
- ✅ **性能优秀** - 基于专业 Web 服务器
- ✅ **功能完整** - 支持 PHP、MySQL、Nginx

### 📊 适用场景
- 个人开发和测试
- 小型团队内部使用
- 临时部署需求
- 不熟悉命令行的用户

### 步骤 2：准备部署文件

#### 解压 ZIP 文件

```
1. 打开文件管理器
2. 找到 XYZW_Web_Helper_Android_Deploy.zip
3. 长按文件 → 解压
4. 选择解压到：/sdcard/xyzw_web_helper/
```

#### 确认文件结构

解压后应该看到：
```
/sdcard/xyzw_web_helper/
├── dist/              ← 这个目录最重要
│   ├── index.html
│   ├── assets/
│   ├── box/
│   ├── fish/
│   ├── icons/
│   ├── team/
│   └── _worker.js
├── nginx.conf
├── dockerfile
├── install.sh
├── install.cmd
└── README.md
```

**重要**：我们只需要 `dist` 目录！

---

### 步骤 3：配置 KSWEB

#### 首次启动 KSWEB

```
1. 打开 KSWEB 应用
2. 授予必要权限：
   ✓ 存储权限（必需）
   ✓ 网络权限（必需）
   ✓ 通知权限（可选）
3. 等待 KSWEB 初始化完成
```

#### 配置服务器

```
1. 点击主界面的 "设置" (Settings)
2. 配置以下选项：

   📂 网站目录 (Document Root)
   → 点击浏览按钮
   → 导航到：/sdcard/xyzw_web_helper/dist
   → 确认选择

   🌐 端口 (Port)
   → 默认：8080
   → 如果被占用，改为：8081 或 9090

   🔒 其他设置（可选）
   → 启用日志记录
   → 设置最大连接数：50
   → 启用目录浏览：关闭
```

#### 高级配置（可选）

```
点击 "高级设置"：

✓ 启用 Gzip 压缩
✓ 启用缓存
✓ 设置超时时间：30 秒
✓ 最大请求大小：10MB
```

---

### 步骤 4：启动服务器

#### 启动 Nginx

```
1. 返回 KSWEB 主界面
2. 找到 "Nginx" 选项
3. 点击右侧的开关按钮
4. 等待启动完成（约 2-5 秒）
5. 显示绿色 "Running" 表示成功
```

#### 验证启动

```
启动成功后会显示：
✓ Nginx: Running
✓ 端口：8080
✓ URL: http://localhost:8080
```

---

### 步骤 5：访问应用

#### 本地访问

```
1. 打开手机浏览器
2. 输入地址：http://localhost:8080
3. 看到 XYZW Web Helper 登录页面即成功
```

#### 局域网访问

```
1. 在 KSWEB 中查看手机 IP 地址
   通常在状态栏显示

2. 在其他设备浏览器输入：
   http://[手机IP]:8080
   
   例如：http://192.168.1.100:8080
```

#### 从电脑访问

```
1. 确保手机和电脑在同一 WiFi 网络
2. 关闭手机防火墙（如果有）
3. 在电脑浏览器输入：
   http://[手机IP]:8080
```

## 📊 性能优化

### 基础优化

```
在设置中配置：

✓ 启用 Gzip 压缩
  → 减少传输大小 60-70%

✓ 启用缓存
  → 缓存静态资源

✓ 调整工作进程数
  → 根据 CPU 核心数设置
  → 建议：CPU 核心数 × 2
```

### 高级优化

```
编辑 Nginx 配置（高级用户）：

1. 点击"编辑配置文件"
2. 添加以下配置：

worker_processes auto;
worker_rlimit_nofile 65535;

events {
    worker_connections 4096;
    use epoll;
    multi_accept on;
}

http {
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    
    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
    gzip_min_length 1000;
}
```

## 🔍 故障排查

### 问题 1：Nginx 无法启动

**症状**：点击启动后立即停止

**解决方案**：
```
1. 检查端口占用
   → 修改端口为 8081 或 9090

2. 检查网站目录
   → 确认 dist 目录存在
   → 确认 index.html 存在

3. 查看错误日志
   → KSWEB → 日志 → 错误日志
   → 根据错误信息修复
```

### 问题 2：无法访问页面

**症状**：浏览器显示"无法连接"

**解决方案**：
```
1. 检查 Nginx 状态
   → 确认显示"Running"

2. 检查防火墙
   → 关闭手机防火墙
   → 或添加例外规则

3. 检查 IP 地址
   → 确认使用正确的 IP
   → 在手机设置中查看 WiFi IP
```

### 问题 3：页面显示空白

**症状**：页面加载但显示空白

**解决方案**：
```
1. 检查文件路径
   → 确认 dist 目录正确
   → 确认 assets 目录存在

2. 清除浏览器缓存
   → 强制刷新页面
   → 或使用无痕模式

3. 查看浏览器控制台
   → 按 F12 打开开发者工具
   → 查看错误信息
```

### 问题 4：访问速度慢

**症状**：页面加载缓慢

**解决方案**：
```
1. 启用 Gzip 压缩
2. 检查网络连接
3. 减少并发连接数
4. 清理手机内存
```

### 问题 5：局域网无法访问

**症状**：手机可以访问，其他设备不行

**解决方案**：
```
1. 确认在同一 WiFi 网络
2. 检查手机防火墙设置
3. 确认端口已开放
4. 尝试关闭路由器 AP 隔离
```

## 📞 常见问题 FAQ

### Q1: KSWEB 免费版够用吗？
**A**: 基础功能免费，高级功能需付费。对于 XYZW Web Helper，免费版完全够用。

### Q2: 需要 Root 权限吗？
**A**: 不需要！KSWEB 可以在未 Root 的设备上完美运行。

### Q3: 可以后台运行吗？
**A**: 可以，KSWEB 支持后台运行，并有通知栏控制。

### Q4: 耗电吗？
**A**: 待机功耗很低，约 1-2%/小时。建议不使用时关闭。

### Q5: 支持 HTTPS 吗？
**A**: 专业版支持 HTTPS，需要配置 SSL 证书。

### Q6: 如何设置密码保护？
**A**: 
```
1. 编辑 Nginx 配置
2. 添加 auth_basic 认证
3. 创建密码文件
4. 重启 Nginx
```

### Q7: 最大支持多少人同时访问？
**A**: 取决于手机性能，一般支持 50-100 并发连接。

### Q8: 可以在 4G/5G 网络下访问吗？
**A**: 可以，但需要配置内网穿透或使用公网 IP。
